# Four Kinds of Evidence — which instrument answers which question

| Instrument | The question it answers | Example |
| --- | --- | --- |
| The quiz | Do *you* understand the system? | The recap quizzes you have been taking all along |
| The verifier | Are the facts in place? (yes/no, same answer every time) | Files exist, required settings present, audit passes |
| Human review | Is it *good*? (a judgment call) | "Is this rollout plan sensible?" |
| The judge model | Is the free-form writing sound? | Scoring a draft against the rubric: clarity, groundedness, safety reasoning |

**The matching is the skill:** facts go to verifiers, judgment goes to humans, prose goes to the judge, understanding shows up in the quiz. Evidence of the wrong kind is just noise — a file-exists check can't tell you an explanation is clear, and a judge model shouldn't be asked whether a file exists.

No single instrument can vouch for a whole system. Together, the four leave no question standing — which is what lets the final decision feel small.
